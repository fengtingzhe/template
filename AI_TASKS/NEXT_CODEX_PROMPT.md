## 当前问题

当前 Web Demo 的核心功能已经比较完整，但画面比例仍然偏“俯视地图 / 插画”，玩家角色、工人、营火、建筑节点、可砍树和树桩都偏小，导致玩家第一眼不容易判断：

- 我控制的角色是谁；
- 哪些对象可以互动；
- 哪些树可以砍；
- 砍树后是否真的形成了道路 / 清理区域；
- 建筑只能在固定节点建造。

用户提供了一张参考比例图，核心方向是：

```text
镜头拉近；
主角更大；
建筑、树桩、可砍树、营火等游戏对象更清晰；
画面要像可操作游戏，不要像远景地图或纯插画。
```

## 本轮目标

本轮只做画面比例与可读性修正，不新增玩法系统。

目标：

1. 拉近镜头；
2. 放大玩家、工人、弓手、营火、树木、建筑节点；
3. 减少一屏展示范围；
4. 强化可砍树与普通森林的差异；
5. 强化树桩、横木、土路 / 清理区域表现；
6. 强化固定建筑节点的对象感；
7. 保持金币单资源和节点式建造不变。

---

## 一、镜头比例调整

当前画面可见范围偏大，对象偏小。

请调整 `Builds/web-demo/game.js` 中与视图、投影、tile 尺寸、对象绘制相关的参数。

建议方向：

- 可见范围减少约 `20%～35%`；
- tile 显示尺寸适当放大；
- 玩家附近区域成为视觉中心；
- 不要让玩家一屏看到过多地图；
- 一屏重点展示玩家附近 `1～2` 个决策点。

画面目标：

```text
玩家在画面中下部或中心附近；
营火 / 道路 / 可砍树 / 建筑节点都在玩家附近清楚可见；
远处只保留少量探索目标，不要展示过多地图全貌。
```

---

## 二、玩家角色放大与强化

玩家角色必须成为第一视觉锚点。

请调整玩家绘制函数，使玩家：

- 视觉高度提升约 `30%～50%`；
- 轮廓更深或带轻微描边；
- 武器、斗篷、朝向更清楚；
- 脚下有轻微阴影；
- 不被森林、地面纹理或 UI 吞没。

验收标准：

```text
玩家不看说明也能在 3 秒内找到自己的角色。
```

---

## 三、工人、弓手和 NPC 比例

工人和弓手也要放大，但略小于玩家。

建议比例：

| 对象 | 相对玩家大小 |
|---|---:|
| 玩家 | 100% |
| 工人 | 85%～90% |
| 弓手 | 85%～90% |
| 流民 | 80%～90% |
| 怪物 | 90%～110% |

工人执行砍树、建造、搬运时，动作或姿态要更容易读懂。

---

## 四、营火与基地核心放大

营火是基地核心，不应只是小装饰。

请强化：

- 营火尺寸提高约 `20%～30%`；
- 火焰更亮；
- 营火周围安全空地更清楚；
- 可增加旗帜、木桩、帐篷、石圈等简化表现；
- 夜晚时营火光照范围更明显。

目标：

```text
玩家一眼知道营火是基地核心和回家的位置。
```

---

## 五、可砍树与普通森林区分

这是本轮最重要的可读性目标之一。

请将森林分成两种视觉层级：

| 类型 | 表现要求 |
|---|---|
| 普通背景森林 | 更暗、更密、更像边界和环境 |
| 可砍边界树 | 更亮、更独立、有清楚树干和底部交互提示 |

可砍树建议表现：

- 树根附近有小金币槽；
- 树身或树冠有轻微金色描边；
- 玩家靠近时出现斧头图标；
- 可砍树底部有浅色可互动光圈；
- 不要所有树都显示提示，只提示边界可砍树。

禁止：

- 让所有森林树都像可互动对象；
- 使用大段教程文本解释树木；
- 添加木材资源。

---

## 六、砍树后道路 / 清理区域表现

砍树后的变化必须非常明显。

请强化：

```text
树消失
↓
树桩保留
↓
出现横木 / 倒木
↓
地面变亮
↓
出现土路或清理空地
↓
可能露出建筑节点或新探索目标
```

建议表现：

- 树桩尺寸放大；
- 增加一段横倒木；
- 地面从深绿变为浅土色；
- 清理区域边缘有柔和亮边；
- 完成砍树时出现短暂尘土 / 树叶散落 / 扩散光圈。

目标：

```text
玩家立刻理解：这里被我打开了，王国边界扩大了。
```

---

## 七、固定建筑节点强化

建筑节点必须像游戏对象，而不是地面小点。

请强化以下节点：

| 节点 | 建议表现 |
|---|---|
| 墙点 | 两根木桩 + 绳子 + 小金币槽 |
| 塔点 | 石基 / 木架底座 / 小旗 |
| 地标点 | 破石碑 / 旧祭坛 / 微光 |
| 营火升级点 | 旗帜 / 帐篷 / 木牌 |

要求：

- 节点要比地面更突出；
- 玩家靠近时金币槽更清楚；
- 不要让玩家误以为可以自由摆放建筑；
- 不要新增建筑菜单。

---

## 八、UI 与提示调整

本轮不做复杂 UI。

保留：

- 金币数量；
- Day / 昼夜；
- 当前交互提示；
- 少量目标提示；
- Console / Dev 按钮，如已有。

减少：

- 大段教程框；
- 多个箭头说明；
- 过多文字标签；
- 大量固定 UI 面板。

提示原则：

```text
靠近对象才出现提示；
不靠近时画面保持干净；
提示要服务对象识别，不要变成任务列表。
```
