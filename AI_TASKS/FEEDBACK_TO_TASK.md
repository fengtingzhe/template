# FEEDBACK_TO_TASK：试玩反馈转任务流程

## 定位

本文件用于把试玩反馈转化为下一轮可执行任务。

它连接：

```text
试玩体验 → 问题分类 → 优先级 → Codex 任务
```

---

```markdown

---

## 转任务原则

P0 反馈优先转成下一轮 Codex 任务。

P1 反馈可以合并到下一轮迭代。

P2 反馈先记录，不抢占当前 Demo 目标。

---

## 常见反馈到任务映射

| 反馈类型 | 典型任务 |
|---|---|
| 看不懂 | 增加目标提示、操作提示、引导文本 |
| UI 不清楚 | 调整信息层级、按钮样式、布局 |
| 反馈不明显 | 增加动画、音效、弹字、状态变化 |
| 数值不合理 | 调整配置表、增加数值模拟 |
| 操作不顺 | 优化输入、点击区域、交互流程 |
| 节奏太慢 | 缩短等待、提高反馈频率、调节目标 |
| 需要新工具 | 写入 `DESIGN_HUB/17_AUXILIARY_TOOLS_PLAN.md` |
| Bug | 生成 Codex 修复任务 |

---

## 输出要求

当 AI 根据试玩反馈生成任务时，必须说明：

```text
反馈是什么；
属于哪类问题；
优先级是什么；
为什么要做；
建议改哪些文件；
如何验收。
```
