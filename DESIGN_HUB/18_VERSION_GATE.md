# 18_VERSION_GATE：版本推进规则

## 定位

本文件用于定义项目从一个版本进入下一个版本的条件。

它的作用是防止 AI 一次性做太多，也防止项目长期停留在模糊状态。

---

## 总体阶段

```text
v0.x = Web Demo 原型阶段
v1.x = Unity 源码学习 / 改造阶段
v2.x = Unity 独立原型阶段
v3.x = 可展示垂直切片阶段
v4.x = 商业化验证阶段
```

---

## v0.x：Web Demo 原型阶段

### v0.1：最小核心操作

目标：

```text
只验证最小核心操作是否成立。
```

要求：

- [ ] 能启动 Web Demo；
- [ ] 玩家能执行一次核心操作；
- [ ] 有最基础的成功 / 失败反馈；
- [ ] 不追求完整 UI；
- [ ] 不追求完整数值。

---

### v0.2：基础 UI 与目标反馈

目标：

```text
让玩家知道自己要做什么、现在状态如何、结果是什么。
```

要求：

- [ ] 有当前目标提示；
- [ ] 有关键状态显示；
- [ ] 有明确操作反馈；
- [ ] 有基础成功 / 失败反馈；
- [ ] 有 Console / Dev 面板。

---

### v0.3：配置化与基础数值

目标：

```text
让策划可以通过配置调整关键参数，而不是改代码。
```

要求：

- [ ] 核心数值来自配置文件；
- [ ] 关卡 / 目标 / 奖励等关键参数可配置；
- [ ] 不大量硬编码策划频繁调整的内容；
- [ ] 配置字段含义清楚。

---

### v0.4：P0 辅助工具

目标：

```text
补上阻塞策划迭代的最小辅助工具。
```

要求：

- [ ] 已完成辅助工具需求分析；
- [ ] 已确认 P0 工具；
- [ ] P0 工具有最小可用版本；
- [ ] 工具能导入 / 导出配置，或明显降低策划修改成本。

---

### v0.5：可评审 Web Demo

目标：

```text
能给美术、程序、制作人或外部评审直观看懂。
```

要求：

- [ ] 有开始说明；
- [ ] 有核心操作提示；
- [ ] 有当前目标；
- [ ] 有关键反馈；
- [ ] 有版本号；
- [ ] 有一键启动方式；
- [ ] 能完成一次核心循环；
- [ ] 已记录试玩反馈。

---

## 进入 v1.x 的条件

进入 Unity 源码学习 / 改造阶段前，必须满足：

- [ ] Web Demo 核心玩法已验证；
- [ ] UI 信息层级基本清楚；
- [ ] 最小数值和反馈已能说明问题；
- [ ] 已记录试玩反馈；
- [ ] 已确认有必要寻找 Unity 源码项目；
- [ ] 已准备进行源码筛选和体检。

---

## v1.x：Unity 源码学习 / 改造阶段

### v1.1：源码候选筛选

- [ ] 至少找到 3 个候选源码；
- [ ] 已检查版本、许可证、依赖、规模、玩法接近度；
- [ ] 已填写 `UNITY_SOURCE/SOURCE_CANDIDATES.md`。

### v1.2：源码跑通

- [ ] 选定一个源码项目；
- [ ] 能用指定 Unity 版本打开；
- [ ] 已记录报错；
- [ ] 已修复必要兼容问题；
- [ ] 能进入主场景或核心流程。

### v1.3：源码结构学习

- [ ] 已整理场景结构；
- [ ] 已整理核心脚本；
- [ ] 已整理 UI / 数据 / 资源组织；
- [ ] 已写入 `UNITY_SOURCE/STRUCTURE_NOTES.md`。

### v1.4：小步改造

- [ ] 完成一个低风险局部改造；
- [ ] 未破坏可运行状态；
- [ ] 已记录改造过程和风险。

---

## 进入 v2.x 的条件

- [ ] 至少跑通一个 Unity 源码项目；
- [ ] 已完成源码结构分析；
- [ ] 已完成若干小步改造；
- [ ] 策划确认需要自建 Unity 原型；
- [ ] 已明确自建原型的最小范围。
