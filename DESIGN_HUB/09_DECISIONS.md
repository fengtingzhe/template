## 已确认通用决策

### [通用] 当前阶段目标是 Demo，不是完整游戏

#### 决策内容

当前阶段目标是快速搭建可试玩、可展示、可验证的游戏 Demo。

#### 决策原因

AI 现阶段完全自动化开发复杂游戏仍有难度。更合理的目标是利用 AI 快速表达核心玩法、界面流程、表现方向和项目可行性。

#### 状态

已确认。

#### AI 执行要求

AI 不得过早实现复杂商业化系统、完整后端、完整 SDK、复杂存档同步或超出 Demo 目标的大型系统。

---

### [通用] 先 Web Demo，后 Unity 源码学习 / 改造

#### 决策内容

项目默认采用：

```text
v0.x = Web Demo 原型阶段
v1.x = Unity 源码学习 / 改造阶段
v2.x = Unity 独立原型阶段
v3.x = 可展示垂直切片阶段
v4.x = 商业化验证阶段
```

Web Demo 用于快速验证核心玩法和界面信息。

Unity 源码学习 / 改造阶段用于学习真实 Unity 工程结构、解决兼容性问题、小步改造参考项目。

#### 决策原因

直接学习完整游戏引擎仍然需要代码基础和工程经验。相比从 0 创建引擎项目，先用 Web Demo 表达玩法，再通过 Unity 源码项目学习真实工程结构，更符合策划主导的 AI 工作流。

#### 状态

已确认。

#### AI 执行要求

AI 不得跳过 Web Demo 阶段直接创建复杂 Unity 工程。

AI 不得默认创建其他引擎工程。

进入 Unity 源码学习 / 改造阶段前，必须参考：

```text
DESIGN_HUB/16_UNITY_SOURCE_WORKFLOW.md
```

### [通用] 不再默认使用 Godot 作为 Demo 主引擎

#### 决策内容

当前模板不再默认使用 Godot 作为 Demo 主引擎，也不默认进入 Godot Demo 阶段。

#### 决策原因

实际体验表明，即使 Godot 较轻量，也仍然需要代码基础、引擎理解、节点系统、场景结构、资源管理和脚本绑定经验，不适合作为当前策划快速上手的默认路线。

#### 状态

已确认。

#### AI 执行要求

AI 不得默认创建 Godot 工程、Godot 场景、GDScript 脚本或 Godot 专用目录。

如未来某个项目明确需要 Godot，必须由人类制作人单独确认，并写入本文件作为项目级决策。

---

### [通用] Console 只默认包含通用开发者工具

#### 决策内容

所有 Demo 默认提供 `Console` / `Dev` 按钮，但只包含通用开发者工具：

```text
音乐开关；
音效开关；
暂停 / 继续；
重置场景 / 重置 Demo；
显示 FPS。
```

开启 FPS 显示后，FPS 数值应显示在游戏画面的右下角。

#### 状态

已确认。

#### AI 执行要求

AI 不得默认加入加资源、跳转时间、生成敌人、解锁节点、跳关等项目特定功能。如项目需要，必须在任务卡中单独确认。
