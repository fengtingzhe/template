# 01_PROJECT_BRIEF：项目定位

## 游戏一句话描述

一款面向泛用户的轻策略探索建造 Demo：玩家孤身进入森林探索未知，用唯一资源“金币”命令村民砍树、建造和升级，把发现的流民、地标和机会逐步纳入王国安全范围，并在夜晚承受扩张带来的防守压力。

---

## 游戏类型

- 主类型：轻策略探索建造游戏
- 副类型：生存防守 / 基地扩张 / 低操作策略 / 单局 Demo
- 参考产品：
  - `Kingdom: New Lands`：参考其低操作、高策略压力、金币单资源、节点式建造、昼夜防守、君主亲自探索的体验。
  - `饥荒`：参考其斜俯视角下对象与地面的分离感、可交互物件的清晰轮廓和游戏画面可读性。
- 不参考产品：
  - 重度 SLG：不做多资源、多队列、长周期行军、重 PVP 压迫。
  - 传统 RTS：不做框选单位、编队、多线微操、高 APM 战斗。
  - 复杂生存建造：不做自由铺路、自由摆建筑、大量材料采集和复杂制造链。

## 目标用户

- 年龄：泛用户，暂不限定年龄段。
- 性别：暂不限定。
- 游戏经验：非 SLG / RTS 核心向玩家；能够接受轻度策略、探索、建造和防守节奏的普通玩家。
- 付费习惯：Demo 阶段不验证付费。
- 游玩场景：短时体验、快速理解、快速获得反馈；适合展示“探索 → 发现 → 建设 → 扩张 → 夜晚压力”的核心循环。

## 产品设计理念

SLG 和 RTS 已经证明，“探索未知、建立据点、扩张领地、从弱变强”的策略幻想在全球有广泛受众。但传统 SLG 策略门槛高，传统 RTS 操作门槛高，主流战争策略又往往把乐趣放在重度扩张和战争上。

本项目要做的是对 SLG 和 RTS 做减法：

```text
保留：探索、建造、扩张、即时压力、策略取舍；
减少：多资源、多系统、多单位微操、复杂科技树、重战争和长期压迫。
```

本项目的核心乐趣放在“探索”上：

```text
玩家先孤身发现未知；
再用金币驱动村民建设；
把探索发现逐步转化为可使用的王国领地；
夜晚用短促压力反馈扩张代价。
```

## 当前 Demo 目标

当前版本只追求：

- 可运行；
- 可试玩；
- 可验证核心体验；
- 可展示核心玩法；
- 可展示主要 UI 流程；
- 可快速迭代。

当前 Demo 重点验证：

- 玩家是否愿意进入森林探索；
- 玩家是否能理解金币是唯一资源；
- 玩家是否能理解“投金币 → 村民执行”；
- 玩家是否能理解砍树的意义是扩大王国控制范围；
- 玩家是否能理解建筑只能建在特定节点；
- 玩家是否能感受到“扩张带来夜晚风险”；
- 画面是否足够干净，核心玩法是否一眼可懂。

当前版本不追求：

- 最终美术品质；
- 完整商业化；
- 完整后端；
- 完整关卡数量；
- 完整上线品质。

---

## 长期版本目标

长期目标是形成一款轻量但有策略压力的探索建造游戏：玩家在不同森林 / 荒野地图中探索、发现、建设、扩张，通过昼夜循环和有限金币做出取舍，逐步把未知区域纳入安全边界。

长期版本可以考虑：

- 更多地形主题；
- 更多地标和探索事件；
- 更多建筑升级层级；
- 更丰富的夜晚威胁；
- 更清晰的单局目标；
- 更完整的移动端体验。

但任何扩展都必须服务“探索优先”的核心理念。

---

## 商业模式方向

Demo 阶段不验证商业化。

后续如进入移动端商业化阶段，可以再评估：

- 买断制；
- 广告激励；
- 轻度内购；
- 章节 / 地图扩展包。

商业化不得破坏核心体验，不得把金币经济改造成强付费压力系统。
