# 17_AUXILIARY_TOOLS_PLAN：辅助工具规划

## 定位

本文件用于记录项目配套辅助工具规划。

辅助工具不属于游戏本体，但用于提升策划、开发、测试、美术协作和内容生产效率。

立项阶段，AI 不应只分析“要做什么游戏”，还应主动分析：

```text
这个项目需要哪些工具，才能让策划更高效地迭代？
哪些内容如果每次都找程序，会严重拖慢项目？
哪些工具适合先用 Web 工具做最小可用版？
```

## 7. 工具开发阶段建议

### Web Demo 阶段

优先放在：

```text
WEB_DEMO/Tools/
```

示例：

```text
WEB_DEMO/Tools/level-editor.html
WEB_DEMO/Tools/balance-simulator.html
WEB_DEMO/Tools/config-validator.js
```

---

### Unity 源码学习阶段

优先放在：

```text
UNITY_SOURCE/Tools/
```

用途：

```text
分析外部源码；
生成配置；
整理报错；
辅助兼容性修复；
辅助源码结构学习。
```

---

### Unity 独立原型阶段

优先放在：

```text
UNITY_PROJECT/Tools/
```

如确实需要 Unity Editor 工具，可以放在 Unity 项目内部：

```text
UNITY_PROJECT/YourUnityProject/Assets/Editor/
```

---

## 8. 工具开发原则

- 先做最小可用版本；
- 优先 Web 工具；
- 优先让策划能独立配置；
- 优先支持导入 / 导出 JSON；
- 如策划常用 Excel，可考虑 Excel / CSV 导入导出；
- 不要过早做复杂后台；
- 不要让工具开发超过游戏 Demo 本体；
- 工具必须服务当前验证目标；
- 工具必须降低沟通成本，而不是增加维护成本。
