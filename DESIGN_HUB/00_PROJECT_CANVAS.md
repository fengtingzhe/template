# 00_PROJECT_CANVAS：项目立项档案

## 定位

本文件是每个新项目的立项入口、想法池和持续记录区。

它用于记录项目最初的想法、ChatGPT 立项访谈结果、后续新增 idea，以及尚未整理进正式设计文档的内容。

本文件不是最终设计定稿。

正式设计应从本文件中提炼到：

```text
DESIGN_HUB/01_PROJECT_BRIEF.md
DESIGN_HUB/02_CORE_GAMEPLAY.md
DESIGN_HUB/03_PLAYER_EXPERIENCE.md
DESIGN_HUB/05_ECONOMY_AND_BALANCE.md
DESIGN_HUB/09_DECISIONS.md
DESIGN_HUB/10_OPEN_QUESTIONS.md
DESIGN_HUB/12_DEMO_SCOPE.md
```

---

## 使用流程

新项目启动时，建议流程如下：

```text
1. 用户开启一个 ChatGPT 项目专属对话；
2. ChatGPT 读取 GitHub 仓库中的模板文件；
3. ChatGPT 重点读取本文件；
4. ChatGPT 按“立项访谈问题清单”向用户提问；
5. 用户回答游戏类型、题材、目标用户、核心玩法、参考产品等问题；
6. ChatGPT 先整理立项档案草案；
7. 用户确认；
8. ChatGPT 再写入本文件；
9. ChatGPT 从本文件提炼正式设计文档；
10. 确认后再进入 Web Demo 开发。
```

重要原则：

```text
先讨论；
再确认；
再写入 GitHub；
最后才进入开发。
```

## 11. ChatGPT 立项访谈问题清单

每次新项目立项时，ChatGPT 应优先询问：

1. 你想做什么类型的游戏？
2. 题材是什么？
3. 玩家扮演谁？
4. 玩家在前 1 分钟主要做什么？
5. 你希望玩家觉得爽、紧张、轻松、策略、探索，还是成长？
6. 有哪些参考产品？
7. 哪些东西明确不想做？
8. 目标用户是谁？
9. 当前最小 Web Demo 要验证什么？
10. 有没有已经想到但还不确定的 idea？
11. 这个项目未来是否可能进入 Unity 源码学习 / 改造阶段？
12. 当前有哪些问题还没想清楚，需要先记录为待决策？
