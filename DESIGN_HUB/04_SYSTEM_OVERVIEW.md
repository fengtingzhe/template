## Demo 阶段系统优先级

```text
核心玩法系统 > UI 流程系统 > 反馈表现系统 > 配置数据系统 > 长期扩展系统
```

当前最优先验证：

```text
玩家能否理解核心操作；
玩家能否完成一次核心循环；
玩家是否能看懂主要 UI；
关键反馈是否清楚；
该玩法是否值得继续做。
```

## 不应过早实现的系统

Demo 阶段通常不应过早实现：

- 完整后端；
- 正式广告 SDK；
- 正式支付；
- 复杂账号系统；
- 复杂存档同步；
- 完整多人系统；
- 大规模关卡编辑器；
- 复杂科技树；
- 大量内容系统；
- 完整商业化系统。

如果 AI 认为需要提前实现这些系统，必须写入：

```text
DESIGN_HUB/10_OPEN_QUESTIONS.md
```
