# 20_DEMO_PRESENTATION_STANDARD：Demo 展示标准

## 定位

本文件用于定义 Web Demo 和后续可展示版本的展示标准。

Demo 不只是“能跑”，还要能让美术、程序、制作人或外部评审快速看懂。

---

## Web Demo 最低展示标准

每个可展示 Web Demo 至少应包含：

- [ ] 开始说明；
- [ ] 核心操作提示；
- [ ] 当前目标；
- [ ] 关键状态显示；
- [ ] 成功反馈；
- [ ] 失败或错误反馈；
- [ ] 重置 Demo；
- [ ] Console / Dev 面板；
- [ ] FPS 显示开关；
- [ ] 版本号；
- [ ] 一键启动方式；
- [ ] 简短 README。

---

## 给美术看的标准

美术需要看懂：

```text
画面风格方向；
主视觉区域；
角色 / 物件 / 地图关系；
哪些对象可交互；
哪些反馈需要特效；
哪些 UI 是核心信息。
```

---

## 给程序看的标准

程序需要看懂：

```text
核心循环；
输入方式；
状态变化；
配置项；
需要迁移到 Unity 的系统；
哪些只是临时表现。
```

---

## 给制作人 / 策划看的标准

制作人 / 策划需要能判断：

```text
玩家是否看得懂；
核心操作是否成立；
节奏是否太慢；
反馈是否足够；
是否值得继续迭代；
是否需要辅助工具。
```

---

## 展示前检查

- [ ] `WEB_DEMO/run_web_demo.bat` 可用；
- [ ] 打开后无需额外配置即可体验；
- [ ] 页面中能看到版本号；
- [ ] 控制台无明显报错；
- [ ] 基础流程可完整走通；
- [ ] 有重置方式；
- [ ] 有说明当前 Demo 不代表最终品质。

---

## 不合格 Demo 特征

```text
只有代码能跑，但玩家不知道要做什么；
没有目标提示；
没有反馈；
没有版本号；
没有一键启动；
没有说明临时资源和正式资源的区别；
必须由开发者口头解释才能理解。
```
