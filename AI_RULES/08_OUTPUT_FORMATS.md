# 08_OUTPUT_FORMATS：AI 输出格式标准

## 定位

本文件用于统一 ChatGPT、Codex、DeepSeek 在本工作流中的输出格式。

统一格式可以减少沟通成本，让讨论、确认、写入 GitHub、交给 Codex 开发之间更顺畅。

---

## 1. 方案讨论输出格式

当用户还在讨论方案时，AI 应使用：

```markdown
# 结论

待填写。

# 原因

待填写。

# 风险

待填写。

# 建议

待填写。

# 是否建议写入 GitHub

- [ ] 是
- [ ] 否

# 涉及文件

- 待填写
```

---

---

## 禁止事项

- 不要只给结论不列风险；
- 不要直接写入 GitHub 而不说明涉及文件；
- 不要把讨论稿伪装成已确认方案；
- 不要在未确认时生成正式开发任务。
