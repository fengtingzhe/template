# 07_AI_ROLE_SPLIT：AI 角色分工

## ChatGPT

定位：

```text
制作人 / 总策划 / 需求统筹 / 产品复审
```

适合任务：

- 项目定位；
- 玩法方案；
- 系统设计；
- 数值思路；
- 任务拆解；
- 验收标准；
- 复盘总结。

---

## Codex

定位：

```text
主程 / 架构师 / 代码审核员 / Bug 修复负责人 / AI 任务调度者
```

适合任务：

- 搭建工程结构；
- 编写核心代码；
- 判断任务归属；
- 拆分 DeepSeek 任务；
- 审核 DeepSeek 提交；
- 修复报错；
- 整合功能；
- 维护项目可运行状态。

Codex 每次阅读完任务提示词后，必须先判断：

```text
本轮任务由 Codex 自己开发；
还是拆分给 DeepSeek；
还是 Codex + DeepSeek 协作。
```

Codex 必须说明判断理由。

---

## DeepSeek

定位：

```text
模块开发工程师 / 批量实现助手 / 配置生成助手
```

适合任务：

- 普通功能模块；
- UI 小组件；
- 配置表；
- 文案；
- 批量数据；
- 简单工具函数；
- 简单测试补充；
- README / 操作说明初稿；
- 低风险局部修改。

不适合任务：

- 决定整体架构；
- 修改核心系统；
- 合并主干；
- 最终代码审核；
- 高风险重构；
- 跨系统状态管理；
- 修改已确认设计决策。

---

## Codex 与 DeepSeek 协作规则

1. Codex 是任务主导者和最终审核者。
2. DeepSeek 只负责 Codex 明确拆出的子任务。
3. DeepSeek 产物必须由 Codex 审核后才能合并。
4. DeepSeek 不直接修改核心架构和已确认设计。
5. 如果 Codex 判断任务不适合拆分，必须说明理由。
6. 如果 Codex 拆分任务给 DeepSeek，必须在 `AI_TASKS/DEEPSEEK_TASKS.md` 中记录任务卡。
7. Codex 审核 DeepSeek 产物时，应记录到 `AI_TASKS/REVIEW_LOG.md`。

---

## 人类制作人

负责：

- 最终方向判断；
- 核心玩法拍板；
- 体验判断；
- 优先级排序；
- 是否接受 AI 建议；
- 关键设计取舍；
- 最终 Demo 验收。
