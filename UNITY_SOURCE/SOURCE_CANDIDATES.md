# SOURCE_CANDIDATES：Unity 源码候选评分表

## 定位

本文件用于记录 Unity 源码学习 / 改造阶段的候选源码项目。

目标是在真正导入和改造源码前，先判断源码是否适合学习、运行和小步改造，避免选错项目导致大量时间浪费。

---

## 使用阶段

```text
v1.1 = Unity 源码候选筛选
```

---

## 评分维度

| 维度 | 分值 | 说明 |
|---|---:|---|
| Unity 版本明确 | 10 | README 或项目文件中能看出 Unity 版本 |
| 可运行性 | 15 | 有运行说明，理论上可打开主场景 |
| 许可证清楚 | 15 | License 明确，允许学习 / 修改 / 参考 |
| 项目规模合适 | 10 | 不过大，不是复杂商业级工程 |
| 玩法接近目标 | 15 | 与当前 Web Demo 或目标项目方向接近 |
| 依赖少 | 10 | 不依赖大量付费插件或复杂外部服务 |
| 代码可读性 | 10 | 脚本结构清楚，命名和组织可理解 |
| 资源完整性 | 5 | 关键资源没有明显缺失 |
| 适合 AI 修改 | 5 | 文本文件清楚，改动范围可控 |
| 学习价值 | 5 | 能帮助理解 Unity 工程结构 |
| 总分 | 100 | 80+ 推荐，60～79 可参考，60 以下不建议 |
